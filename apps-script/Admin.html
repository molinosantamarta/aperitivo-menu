Come funziona Menumal

Architettura del sistema

  • Menumal e il pannello operativo: qui modifichi prodotti, prezzi, descrizioni, visibilita, disponibilita e immagini semplici.
  • Google Sheet e il database sorgente del menu. I tab usati dal sistema sono `admin_sections`, `admin_items`, `admin_settings` e `admin_audit_log`.
  • Apps Script e il backend di Menumal: legge e scrive il Google Sheet, espone il JSON live del menu e gestisce la logica admin.
  • Agri-Menu e il menu pubblico aperto dai clienti via QR code. Legge l endpoint live di Apps Script e applica i dati ai layout del sito.
  • GitHub contiene il codice del menu pubblico e anche i file sorgente di Menumal. Serve per modifiche grafiche, strutturali o di logica.

Dove vivono i dati

  • Sezioni del menu: stanno in `admin_sections`.
  • Prodotti del menu: stanno in `admin_items`.
  • Impostazioni runtime: stanno in `admin_settings`.
  • Storico operazioni: sta in `admin_audit_log`.
  • Il menu pubblico non usa piu il vecchio `database-semplice` e non usa piu il fallback CSV. La sorgente live e l endpoint JSON di Apps Script.

Flusso di una modifica fatta da Menumal

  • Apri un prodotto esistente oppure crei un nuovo prodotto.
  • Menumal invia i dati ad Apps Script.
  • Apps Script salva il record nel Google Sheet.
  • Se carichi un immagine da file, Apps Script la salva su Google Drive e scrive nel foglio il riferimento finale dell immagine.
  • Dopo il salvataggio, Menumal rilegge il Google Sheet e verifica anche il payload live del menu.
  • Se tutto va bene, il cliente che apre Agri-Menu via QR vede i dati aggiornati senza deploy GitHub.

Apri il payload live

Che cosa cambia davvero per il cliente

  • Se un prodotto e visibile, il cliente lo vede nel menu.
  • Se un prodotto e nascosto, il cliente non lo vede, ma il record resta nel database.
  • Se un prodotto e disponibile, il cliente lo vede come ordinabile.
  • Se un prodotto e in arrivo o non disponibile, il cliente lo vede con quello stato.
  • Le card legacy continuano a usare la loro struttura grafica del sito, ma i contenuti principali possono essere aggiornati da Menumal.

Quando usare Menumal e quando serve codice

  • Usa Menumal per aggiungere prodotti semplici, cambiare prezzo, titolo, descrizione, visibilita, disponibilita e immagine.
  • Serve il codice per layout speciali, card legacy personalizzate, animazioni, carousel, cambi CSS o cambi di logica frontend.
  • Se una modifica riguarda solo il contenuto di un prodotto, in genere non serve pubblicare GitHub.
  • Se una modifica cambia l aspetto o il comportamento del menu pubblico, serve intervenire nella repo e poi pubblicare.

Preview, deploy e manutenzione

  • La preview locale di Menumal serve per provare la UI prima di aggiornare il progetto Apps Script reale.
  • Quando cambiano i file di Menumal, bisogna aggiornare `Code.gs`, `Admin.html`, `AdminStyles.html`, `AdminScripts.html` e, se necessario, `appsscript.json`, poi creare una New version in Apps Script.
  • Quando cambia Agri-Menu pubblico, il deploy passa da GitHub Pages.
  • Questa sezione Schema sistema va tenuta aggiornata ogni volta che cambiano flusso dati, tab del foglio, endpoint live, logica immagini o processo di deploy.
